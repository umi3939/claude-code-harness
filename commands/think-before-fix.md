# Think Before Fix フロー

解析(analyzer)またはレビュー(reviewer)で問題が検出された後の修正手順。
表面的な修正に飛びつかず、thinkerで根本原因を考えてから修正すること。

## 手順

### 1. 検出された問題を確認する
解析/レビュー結果から、MED以上の問題・指摘事項を一覧化する。
- 問題の重要度（HIGH/MED/LOW）を確認
- 複数の問題がある場合は依存関係を整理

### 2. thinkerで根本原因を分析する
```
Agent(subagent_type="thinker")で以下を問う:
- この問題の根本原因は何か？
- 表面的な症状と構造的な原因を区別する
- 修正が他の箇所に波及する影響範囲は？
- 同じパターンの問題が他にもないか？
```
thinkerの結果をSTMに記録する（以降のステップへの残差接続）。

### 3. implementerに修正を委任する
thinkerの分析結果を踏まえて、implementerに具体的な修正を指示する。
- 根本原因に対する修正であることを確認
- 表面的な対症療法ではなく構造的な修正を指示
- TDD: テストを先に書いてから修正

### 4. 再検証する
修正後に再度analyzer/reviewerで問題が解消されたことを確認する。
- analyzerで新たなリスクが発生していないか検証
- reviewerで修正品質を確認
- 問題が残っていればStep 2に戻る
