# 並列レビュー（Large Tier限定）

Large tier変更（10+ファイル or 500+行）に対し、3つのreviewerを並列起動して品質を多角的に検証する。

## 前提条件
- フローティアがLargeであること
- post-impl analysisが完了していること
- TeamCreateが実施済みであること

## 手順

### 1. 変更規模の確認
```
git diff --stat HEAD~N  # 変更対象コミット数分
```
10ファイル未満かつ500行未満なら通常の単一reviewerを使う。

### 2. 3つのreviewerを並列起動

**Security Reviewer:**
- 観点: 脆弱性、インジェクション、認証、機密情報漏洩
- プロンプト: 「CLAUDE.mdを全文読んでください。セキュリティ観点でレビューしてください。対象: [ファイルリスト]。チェック項目: コマンドインジェクション、パストラバーサル、機密情報ハードコード、入力検証不足、認証/認可バイパス」

**Performance Reviewer:**
- 観点: パフォーマンス、メモリ、I/O、タイムアウト
- プロンプト: 「CLAUDE.mdを全文読んでください。パフォーマンス観点でレビューしてください。対象: [ファイルリスト]。チェック項目: N+1問題、メモリリーク、ブロッキングI/O、タイムアウト未設定、無限ループリスク」

**Quality Reviewer:**
- 観点: コード品質、テスト網羅性、デッドコード、命名
- プロンプト: 「CLAUDE.mdを全文読んでください。コード品質観点でレビューしてください。対象: [ファイルリスト]。チェック項目: テスト網羅性、デッドコード、命名規則、エラーハンドリング、ドキュメント」

### 3. 結果統合
リーダーが3つの結果を統合し、重複を除去して優先度順に並べる:
- HIGH: 即座に修正（thinker→implementer）
- MEDIUM: 修正必須（implementer）
- LOW: 改善推奨（次サイクルに積む）

### 4. 修正後の再検証
MED+の修正後は再度reviewerを実行（通常の単一reviewerで可）。

## 注意
- Small/Medium tierでは使わない（オーバーキル）
- --worktreeオプションで別コンテキストでの実行を推奨（提案6）
- 3つのreviewerの結果が矛盾する場合はthinkerで判断
