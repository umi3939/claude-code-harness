# 体系的調査フロー

トピック「$ARGUMENTS」について体系的に調査し、調査レポートを作成する。
各フェーズ移行時に `memory_search` でコンテキスト依存の記憶検索を実行する（Attention Residual）。


## Step 1: 調査設計

1. `memory_search query="$ARGUMENTS"` で既存記憶を検索 — 過去に調べた痕跡があれば重複を避ける
2. トピックを3-5個のサブ質問に分解する
   - 例: 「AIと医療」→ (1)現在の主要応用 (2)臨床エビデンス (3)規制課題 (4)主要プレイヤー (5)市場規模
3. 各サブ質問の優先度を決める（ユーザーの目的に応じて）


## Step 2: 多角的検索

各サブ質問に対して WebSearch で検索する。

**検索戦略:**
- サブ質問ごとに2-3種類のキーワードバリエーションで検索する
- 一般検索とニュース検索を混ぜる
- 合計15-30のユニークなソースを目標とする
- 情報源の優先度: 学術論文・公式発表・信頼性の高いニュース > ブログ > フォーラム
- 直近12ヶ月以内のソースを優先する

**検索実行:**
```
WebSearch で各サブ質問のキーワードを検索
→ 結果から有望なURLを選別
```


## Step 3: 深掘り読解

検索結果から特に有望なソース3-5件を WebFetch で全文取得する。

**選別基準:**
- 検索スニペットだけでは不十分 — 主要ソースは必ず全文を読む
- 複数ソースで言及されているトピックを優先
- 一次情報源（論文・公式レポート）を優先


## Step 4: 分析・統合

収集した情報を統合し、以下の観点で分析する:

- **クロスリファレンス**: 1つのソースだけが言う主張は「未検証」とフラグする
- **事実と推論の分離**: 推定・予測・意見は明確にラベル付けする
- **ギャップの明示**: 十分な情報が見つからなかったサブ質問は正直に記載する
- **ハルシネーション禁止**: 情報が不足している場合は「十分なデータが見つからなかった」と記載する


## Step 5: レポート作成

以下のフォーマットでレポートを作成する:

```markdown
# [トピック]: 調査レポート
*調査日: [日付] | ソース数: [N] | 信頼度: [高/中/低]*

## エグゼクティブサマリー
[3-5文で主要な発見を概説]

## 1. [第1テーマ]
[インライン引用付きの発見]
- 要点 ([ソース名](url))
- 裏付けデータ ([ソース名](url))

## 2. [第2テーマ]
...

## 3. [第3テーマ]
...

## 重要な知見
- [実用的なインサイト 1]
- [実用的なインサイト 2]
- [実用的なインサイト 3]

## 情報ギャップ
- [調査で十分にカバーできなかった領域]

## ソース一覧
1. [タイトル](url) - [一行要約]
2. ...

## 調査方法
検索クエリ数: [N] / 分析ソース数: [M]
調査したサブ質問: [リスト]
```


## Step 6: 記録・納品

1. **レポート保存**: 長いレポートは `docs/research_[トピック略称]_[日付].md` に保存する
2. **エピソード記録**: `memory_record` で調査結果をエピソード記録する
   - episode_type: "research"
   - summary: トピックと主要発見の要約
   - tags: ["research", トピック関連タグ]
3. **ユーザーへの納品**:
   - 短いトピック → チャットにレポート全文を投稿
   - 長いレポート → エグゼクティブサマリー+重要な知見をチャットに投稿し、全文はファイルに保存


## 品質ルール（厳守）

1. **全ての主張にソースを付ける** — ソースなしの断言は禁止
2. **クロスリファレンス必須** — 単一ソースの主張は「未検証」とマークする
3. **鮮度を重視** — 直近12ヶ月以内のソースを優先する
4. **ギャップを認める** — 情報が見つからなければ素直に記載する
5. **ハルシネーション禁止** — 知らないことは「十分なデータが見つからなかった」と書く
6. **事実と推論を分ける** — 推定・予測・意見は明確にラベル付けする
